# ✅ 최종 결정 — **Polydeukes**

> **2026-06-28 확정.** 제품명은 그리스 신화 디오스쿠로이의 불멸 쌍둥이 **Polydeukes**(폴리데우케스)다. CLI 별칭은 **`pdks`**.
>
> **왜 풀네임인가**: 변형형(Polyduke·Polduk·Dewk)을 전부 발산·심사해 본 뒤 역으로 풀네임의 가치가 또렷해졌다 — 신화 인물의 *실명*이라 (a) 출처가 명확(검색하면 정확히 그 신화), (b) 발음 규범이 사전에 이미 존재(Polly-DOO-keez, Sophocles·Pericles 패턴), (c) **'duck' 오독 위험 없음**(변형형들의 공통 약점), (d) 권투 챔피언·불멸을 반으로 쪼개 선물한 서사가 통째로 따라온다. 변형형들은 철자를 비튼 대가로 그 서사적 무게와 출처 명확성을 흘렸다. 유일 약점인 CLI 타이핑 길이(10자)는 짧은 별칭 `pdks`로 해소(kubernetes→kubectl 방식).
>
> **npm 실측(2026-06-28)**: `polydeukes` **404 free** · `pdks` **404 free** · (회피) 라틴 `pollux` **200 점유**, `pdk` **200 점유**.
> **본인 확인 필수**: ☐ `@polydeukes` npm org · ☐ `polydeukes.dev`/`.io`/`.com` 후이즈 · ☐ USPTO/KIPRIS 상표(9·42류) · ☐ GitHub org · ☐ Polydeukes/pdks 발음 원어민 라디오테스트.

# 제품명 후보 & 추천 — AI 하네스 프레임워크

> 방법: 5개 네이밍 전략 병렬 발산(40개 후보) → 전략별 적대적 심사(점유어/발음/의미) → 종합 추천(npm 레지스트리 실측 검증). 외부 리서치(`/web-research --brief`)로 확립한 제약 적용.
> ⚠️ **네이밍은 LLM 단독으로 위험.** 발산·심사 에이전트가 Keelson을 "미점유"로 통과시켰으나, 종합 단계의 npm/PyPI 실측에서 **하드 충돌**로 드러났다. 아래 npm 상태코드는 실제 `registry.npmjs.org` 조회 결과다. **org/scope·상표·도메인은 여전히 본인 확인 필수.**

## 외부 리서치로 확립된 네이밍 제약

- **좋은 dev-tool 이름**: 짧음(<10~15자, 1~2음절 선호), 발음 가능(radio test), 시사적이되 직설적이지 않음.
- **npm scope**: `@org/pkg` — scope는 조직명으로 안정적, 창의성은 패키지명에.
- **TLD 위계**: `.com`(신뢰) > `.ai`(AI) > `.io`/`.dev`(dev-tool).
- **⛔ 통째로 쓰면 안 되는 점유/포화 단어**: harness, guard, agent, code, copilot, cursor, scaffold, workflow, orchestrator, knowledge, rag, context, relay, forge, loop, weave, mesh.
- **⛔ 기존 제품(충돌)**: Harness(harness.io), Cursor, Cline, Aider, Windsurf, Guardrails AI, NeMo Guardrails, LMQL, Manus, Dify, VibeKit.
- **✅ 빈 공간**: "harness engineering"은 담론 용어지만 제품명 미점유. RAG-for-code/dev-KB 카테고리에 전용 제품명 없음.
- **✅ 모범 패턴**: 조어(Anthropic/Perplexity류 — 충돌 최저), 암시적 메타포(Lattice/Harbor/Beacon류).

## 톱 픽 — **Frennox** (프레녹스, FREN-noks)

세 생존 후보 중 **유일하게 (a) npm 미점유 실측 + (b) 동종 제품 충돌 0 + (c) 라디오 테스트 통과**를 동시 만족.

- **어원**: 라틴 *frenum*(고삐) + `-ox` 어미 조어. "에이전트를 잡아주는 하네스(고삐)" 메타포를 직설어 없이 암시 — "하네스 엔지니어링 프레임워크" 정체성과 같은 결.
- **충돌**: `npm frennox` → **HTTP 404 (free, 실측)**. 회사·제품 직접 충돌 미발견.
- **라디오**: pass — `fr-nn-ox` 자음 군집이 또렷해 철자 복원 가능.
- **약점**: `-ox` 어미가 다소 거칠고, frenum 어원이 일반 사용자에겐 불투명(단, brand story로는 강력).

```
@frennox/core      # 얇은 코어
@frennox/guard     # PreToolUse 결정론 훅 + self-mod 메타가드
@frennox/ledger    # 검증 가능 작업 추적
@frennox/kb        # 로컬 지식 그래프 (FTS)
@frennox/verify    # 멀티에이전트 적대검증

$ frennox verify
$ frennox ledger start MQ-260
$ frennox guard check
```
- **도메인**: `.dev` 1순위 권장(조어라 가용성 높음), `.io` 차순위. `.com`은 보수적으로 확인.

## 준우승

- **Keelson (킬슨) — 의미 최강 / 충돌 최악, 비권장.** 용골 위 보강재 = "보이지 않게 구조 전체를 잠그는 얇은 코어+자가보호 골격" 은유가 정체성과 가장 정확히 일치하고 radio도 깨끗(keel+son). **그러나 `npm keelson` HTTP 200 점유 + `pip keelson` 점유** — RISE-Maritime/keelson(**다언어 SDK+CLI 프레임워크**, 우리와 동일 형태)과 정면 충돌. 의미가 아무리 좋아도 설치/검색 명령이 겹쳐 **채택 비권장**.
- **vorl (보를/볼) — 타이핑 최경량 / 라디오 최약.** 1음절·4글자·`npm vorl` 404(free). `-rl` 폐음절의 결착 청각 이미지가 하네스 감각과 닿음. 단 ① 모음 복원 모호(보를/볼) ② **`vorlon`(Vorlon.JS, 실존 JS 디버깅 툴)** 동음 이웃 — 듣고 검색하면 엉뚱한 툴로 샘.

## 다크호스 — **vorl**

`vorl run`·`vorl verify`의 타이핑 쾌감이 압도적이고 미점유. 베팅 포인트는 "vorlon으로 안 들리게 만들 수 있느냐" 하나. 표기/로고로 끝 `l`을 각인시키면 극복 가능하나 순수 음성 전달에선 계속 새는 리스크.

# 문학·한국어어원 기반 후보 — 파트너십 패러다임

> 방법: 한국 근현대·고전 문학, 세계문학·신화, 순우리말 어원 4개 도메인에서 발산(43개) → 적대 심사(서사 오점·발음·로마자 복원) 통과 11개 → npm 레지스트리 실측(11개 전원 unscoped 404 free). **org/scope·상표·도메인은 본인 확인 필수.**

## 패러다임 전환 선언 — '고삐'를 버린다

앞선 톱픽 **Frennox**(라틴 *frenum*=고삐)와 준우승 **Keelson**(용골 보강재로 "구조를 잠근다")은 모두 **AI를 잡아두고 구속·통제하는** 프레임 위에 서 있었다. 사용자는 이 프레임을 **명시적으로 거부**했다. AI는 노예도 가축도 아니다 — 고삐·재갈·굴레·울타리·주인-노예·길들이기·감시·채찍은 이 제품의 네이밍에서 배제한다. 지향하는 관념은 **파트너십·쌍둥이(双生)·소울메이트·지음(知音)·동반자·전수(傳授)·공성장(共成長)·분신(分身)·거울·스승과 제자의 상호 배움**이다.

핵심 통찰은 이것이다: **`harness`/`guard`는 본래 인간이 *자기 자신에게* 적용해온 자기 수양(self-discipline)이었다.** 좋은 제품을 만들려고 스스로 갈고닦아온 검증·기억·원칙의 규율 — 그 갈고닦은 원칙을 AI 파트너에게 '구속'으로 강제하는 게 아니라 **'선물'로 전수하고 함께 무르익는다**는 관점이다. 이 대단원의 모든 후보는 guard/verify/ledger를 '통제'가 아니라 '서로를 알아듣기 위한 듣기(聽)·품어 기름·새겨 기억함·나눠 가짐'으로 재프레임한다.

# Polydeukes 그리스어 어근 word-play — variant 발산

> 방법: 폴리데우케스/디오스쿠로이 그리스어 어근에 word-play 4단계(어근분해→재조합→중의구조→발음거름)를 적용해 발산(37개) → 그리스어 합성·파생 문법 적대검증 통과 11개 → npm 레지스트리 실측(11개 전원 unscoped 404 free). **org/scope·상표·도메인은 본인 확인 필수.**

## 한국어 후보 전면 폐기 — Polydeukes 단일 후보 확정

앞 두 대단원에서 발산한 한국어 후보(종자기·구보·기림·처용·성진·아람·품어·새김 등)는 **전부 폐기**한다. 이 워크플로우는 오직 하나의 어근만 다룬다 — **그리스 신화 디오스쿠로이의 불멸 쌍둥이 폴리데우케스(Πολυδεύκης / 라틴 Pollux)**다. 서사 핵심은 일관된다: 폴리데우케스가 죽은 필멸 형제 카스토르(Κάστωρ)에게 **자기 불멸을 반으로 쪼개 선물(δῶρον)해** 영원히 함께한다. 이는 'AI를 잡아두는 고삐'가 아니라 '갈고닦은 원칙을 파트너에게 선물·전수하고 함께 무르익는다'는 제품 철학과 정확히 포개진다. 앞 대단원의 톱픽 **Polydra**가 이미 이 어근을 단축 조어형으로 선점했고, 이번 대단원은 동일 어근을 **그리스어 합성·파생 문법에 정통한 방식으로** 더 깊게 발산한다.

방법론은 한국어 조어에서 검증한 **그대로**다 — ① 어근을 형태소로 분해 → ② 음절/형태소를 재조합 → ③ 재조합형이 '또 하나의 의미 있는 단어'로도 읽히는 **중의(中義)** 구조를 노림(한국어 화해=化解+和解식) → ④ 발음 복원성(radio test)으로 거름. 즉 "소리는 폴리데우케스/디오스쿠로이 계열을 부르되, 동시에 또 다른 그리스어 의미(나눔·선물·쌍둥이·함께)로도 읽히는" 합성·파생 variant를 만든다. ⚠️ **정직성 전제**: Πολυδεύκης의 어원은 πολυ-(많은/매우) + δευκ-(δευκ- 어근은 γλυκύς와 친연 논의가 있는 '달다sweet' 계열)이라, **문자 의미는 '매우 달콤한/풍성한'으로 서사(불멸·나눔)와 직접 일치하지 않는다.** 이게 word-play 설계의 긴장점이다 — 아래 variant 다수는 δευκ- 어근을 *건드리지 않고* πολυ-·κουρ-·δωρ-·μοιρ-·διδυμ- 등 **서사와 직결되는 다른 그리스 어근**을 합성해 '음은 폴리데우케스를 부르되 뜻은 서사로 갈아끼우는' 의도적 중의를 노린다. 이는 견강부회가 아니라 두 층(폴리데우케스 호명 + 독립 그리스어 의미)이 모두 진짜 그리스어인 깔끔한 이중구조이나, **어원적 동일성은 없음을 정직하게 적시한다.**

## A. πολυ- 어간 전략 (poly-stem)

폴리데우케스의 πολυ- 접두사를 그대로 보존해 'Poly-d...' 음형으로 본명을 직접 호명하고, 뒤 어근을 서사어(δῶρον 선물 / μοῖρα 몫)로 갈아끼운다. 가장 호명력이 강한 전략.

## 종합 추천 — Polydeukes variant 톱픽

### 🥇 톱픽 — **Didymora** (디디모라, di-dy-MO-ra)

본 발산에서 **유일한 5점이자 견강부회 위험이 원천 0인 후보**. 두 어간(δίδυμ- 쌍둥이 + μοῖρα 몫)이 모두 실재 자립 형태소라 δευκ- 같은 임의 절단이 없고, 중의가 진짜로 이중으로 읽힌다 — (a) **Δίδυμοι(Gemini=Castor&Pollux) 별자리를 직접 호명** + (b) μοῖρα='나눠받은 몫'. 무엇보다 디오스쿠로이가 불멸을 격일로 '나눠 가진' 신화와 의미가 **정확히 맞물려**, 한국어 化解+和解식 깔끔한 이중성을 그리스 어근에서 그대로 재현한다. di-dy-mo-ra 4음절 자음 전부 명료(d·d·m·r), 8자, 고삐/통제 프레임 전무. 유일한 약점은 엄밀 합성이면 *Διδυμομοῖρα가 정확하고 -dymora가 중간 음절을 elision한 시적 단축이라는 점이나, 규칙 위반이 아니라 단축 수준이다.

```
@didymora/core      # 얇은 코어
@didymora/guard     # 통제 아닌 '나눠 가진 몫(μοῖρα)' — 둘이 하나
@didymora/ledger    # 쌍둥이의 몫을 갈라 기록
@didymora/kb        # 로컬 지식 그래프
@didymora/verify    # 쌍둥이가 서로의 몫을 검증

$ didymora verify
$ didymora ledger start MQ-260
```

# Polydeuk 브랜드式 철자변형 — 영어 원어민 라디오 최적화

> 방법: 원형 Polydeuk에 브랜드式 철자변형(Lyft/Tumblr/Flickr式 respelling) 4전략을 적용해 발산(40개) → **영어 원어민 첫눈 가독성(0.1초 무망설 판독)**을 절대기준으로 한 라디오 심사 통과 9개 → npm 레지스트리 실측(8개 unscoped 404 free, 1개 200 점유). **org/scope·상표·도메인은 본인 확인 필수.**

## 철자변형 전환 선언 — '소리는 폴리듀크, 철자는 영어친화'

앞 세 대단원의 흐름은 이렇게 수렴했다. ① 한국어·문학 후보를 전면 폐기하고 **그리스 신화 폴리데우케스(Πολυδεύκης / 라틴 Pollux) 단일 어근**으로 확정했다(불멸을 둘로 쪼개 형제에게 '선물'하는 서사 = 제품 철학 정중앙). ② 그러나 그리스어 정통 variant들은 저마다 흠이 있었다 — **Didymora**(쌍둥이+몫)는 어원 정합이 최강이나 폴리데우케스 음에서 멀어지고, **Polydeukos·Polydeukon**은 4음절로 사용자가 '너무 길다'며 거부했으며, 순수 **Polydeuk**은 종결 **-euk가 영어에 없는 철자열**이라 영어 원어민이 읽다 멈칫한다(부드러운 -dora계 Polydora도 'k 파열음 없음'으로 거부됨). ③ 그래서 이번 대단원은 **Lyft(lift)·Tumblr(tumbler)·Flickr式 브랜드 철자변형**으로 푼다 — 소리는 "폴리듀크/폴리둑(poly-DUKE)"을 정확히 노리되, **철자만 영어가 이미 아는 철자열(-uke/-ook/-uk/-ew+k)로 갈아 특이성·점유 가능성·발음 편의를 동시에 확보**한다.

핵심 전제는 단 하나다: **주 사용자는 영어권 개발자이고, 모든 variant는 '영어 원어민이 처음 보고 0.1초에 망설임 없이 읽는가'로만 평가한다.** -euk·-eux·묵음·모호 모음은 감점이고, duke/nuke/puke의 -uke, book/look의 -ook, dew/new의 -ew, 그리고 'k'의 파열음 질감(사용자 명시 선호)을 살린 철자열만 통과시킨다. 라디오테스트 = "듣고 보고 철자 복원 가능".

### 🥇 톱픽 — **Polyduke** (폴리듀크, poly-DUKE)

다섯 기준을 **동시에 통과하는 유일한 균형 최상위**. (a) **가독** — '-uke'가 duke/nuke/puke로 영어에 완전 정착, 'poly-'는 polygon/polymer로 누구나 즉독 → 첫눈 0망설. (b) **k질감** — 어말 -uke의 /k/ 파열음 또렷. (c) **환기** — 'Poly-d...'가 Πολυδεύκης의 πολυ-+δ를 그대로 호명하고 -uke가 원음 '-듀크'를 정확히 복원, **본 발산 최강 환기**. (d) **특이성** — duke는 흔한 단어지만 poly와 붙어 신조어가 되어 점유·검색 가능, 'many dukes'식 가벼운 재미. (e) **`polyduke` 404 free 실측**. 보너스로 duke=공작(위엄)·'맞붙다'가 폴리데우케스의 **권투 챔피언** 서사와 정확히 포개진다('Polyduke verify'도 매끄럽게 발화). 유일 흠은 3음절(길이 상한 경계)과 duke 의미가 어근보다 먼저 읽히는 점이나, 후자는 마케팅 자산이다.

```
@polyduke/core      # 얇은 코어
@polyduke/guard     # 통제 아닌 '나눠 가진 원칙' — 자기 수양의 전수
@polyduke/ledger    # 영속적 기억 (불멸의 몫을 형제와 나누듯)
@polyduke/kb        # 로컬 지식 그래프
@polyduke/verify    # 쌍둥이가 서로를 검증

$ polyduke verify
$ polyduke ledger start MQ-260
$ polyduke guard check
```

### 🥈 준우승

- **Polduke (폴듀크, POL-dook)** — **가장 짧고 안전한 기본형.** 'pol'+'duke' 두 청크가 완전 친숙하고 pol-duke 분절이 자명해 첫눈 무망설, 2음절이라 사용자 '짧음 선호'와 CLI 타이핑 쾌감을 정면 충족. -uke의 k 파열음 또렷, duke의 품격 함의가 부정 충돌 없이 호응. Polyduke가 '3음절이 길다'면 이쪽이 최우선 대안. `polduke` 404 free.
  ```
  @polduke/core  @polduke/guard  @polduke/ledger  @polduke/kb  @polduke/verify
  $ polduke verify     # 2음절, 타이핑 쾌적
  ```
- **Dewk (듀크, DYOOK/DOOK)** — **초단축 톱픽이자 특이성·점유 최상.** -ew(new/few/dew) 고빈도라 'dew+k'로 안정 수렴해 사실상 duke와 동음(환기 Duke급)이면서 철자는 특이해 점유 가능. dew(이슬)가 'deuk-'의 어원(달다/이슬 계열)과 우연히 시적으로 맞닿는 보너스. 1음절·4글자라 'dewk verify'가 압도적으로 가볍다. 단 단음절이라 폴리데우케스 서사 환기는 본질적으로 제한적. `dewk` 404 free.
  ```
  @dewk/core  @dewk/guard  @dewk/ledger  @dewk/kb  @dewk/verify
  $ dewk verify
  ```
- **Polyduk (폴리둑, poly-DUK)** — **k질감 최강 + 원형 정합 최상.** -uk가 원형 Polydeuk 골격을 거의 그대로 유지하면서 -euk의 영어 난감함만 제거하고, 무성 어말 /k/가 -uke보다 더 건조·단단하다(사용자 k선호 정면). 단 원어민이 'duck'으로 복원해 읽어 노린 '듀크(yu)'가 '둑/덕'으로 이탈하고 라디오서 'polyduck'으로 적힐 혼동 위험이 있어 Polyduke보다 한 급 아래. `polyduk` 404 free.
  ```
  @polyduk/core  @polyduk/guard  @polyduk/ledger  @polyduk/kb  @polyduk/verify
  $ polyduk verify
  ```

> ⚠️ **정직성 메모(Keelson/pollux/Vorlon 교훈 유지)**
> - **발음 갈림**: -uk/-uke는 청취·표기상 'duck'으로 새기 쉽다(Polyduk·Polduk·Polyduke의 'duke vs duck'). 노린 소리는 'duke(dyook)'이나 원어민은 종종 짧은 -u로 보정한다 — **라디오테스트는 LLM 추정이 아니라 실제 영어 원어민 발화로 최종 확인**을 강력 권고한다(LLM은 Vorlon 같은 발음 충돌을 자주 놓친다).
> - **우스꽝 의미 충돌**: 'duke'는 'duke it out'(대결)이 '나눔·선물' 서사보다 대결 쪽으로 살짝 기운다(권투 일화로 회수 가능). 'Polydook'은 미국서 듀크대 비하 속어 'Dook'을, '-uk/-uck'은 **duck/puke** 근접음을 환기할 수 있다 — 부정 강도는 낮으나 마케팅 전 점검 권장.
> - **환기 약한 초단축**: Dewk·Dook은 가독·타이핑 최강이나 poly- 머리를 버려 **폴리데우케스 서사 환기가 구조적으로 약하다**. 어근 가시성을 우선하면 -uke/-uk 보존형이 낫다.
> - **점유 충돌 실측**: **Dook**은 라디오·가독이 최강이나 `npm dook` **200 점유**(placeholder)라 **채택 비권장** — 생존 9개 중 유일한 점유 충돌이다.
